# Go 语言设计模式实现

本目录包含了 23 种经典设计模式的 Go 语言实现示例，按照 GoF 设计模式分类。

## 创建型模式 (Creational Patterns)

### 1. 抽象工厂模式 (Abstract Factory)

- **文件**: `abstractfactory/main.go`
- **描述**: 提供一个创建一系列相关或相互依赖对象的接口，而无需指定它们具体的类
- **应用场景**: 跨平台 UI 组件创建、数据库驱动选择

### 2. 建造者模式 (Builder)

- **文件**: `builder/main.go`
- **描述**: 将复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示
- **应用场景**: 复杂对象构建、SQL 查询构建器、配置对象创建

### 3. 工厂方法模式 (Factory Method)

- **文件**: `factory/main.go`
- **描述**: 定义一个创建对象的接口，让子类决定实例化哪一个类
- **应用场景**: 对象创建、插件系统、日志记录器

### 4. 原型模式 (Prototype)

- **文件**: `prototype/main.go`
- **描述**: 用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象
- **应用场景**: 对象克隆、配置模板复制

### 5. 单例模式 (Singleton)

- **文件**: `singleton/main.go`, `singleton/sg/sg.go`
- **描述**: 保证一个类仅有一个实例，并提供一个访问它的全局访问点
- **应用场景**: 数据库连接池、日志记录器、配置管理器

## 结构型模式 (Structural Patterns)

### 6. 适配器模式 (Adapter)

- **文件**: `adapter/main.go`
- **描述**: 将一个类的接口转换成客户希望的另一个接口
- **应用场景**: 第三方库集成、遗留系统接口适配

### 7. 桥接模式 (Bridge)

- **文件**: `bridge/main.go`
- **描述**: 将抽象部分与它的实现部分分离，使它们都可以独立地变化
- **应用场景**: 跨平台开发、驱动程序设计

### 8. 组合模式 (Composite)

- **文件**: `composite/main.go`
- **描述**: 将对象组合成树形结构以表示"部分-整体"的层次结构
- **应用场景**: 文件系统、UI 组件树、组织架构

### 9. 装饰器模式 (Decorator)

- **文件**: `decorator/main.go`
- **描述**: 动态地给一个对象添加一些额外的职责
- **应用场景**: 功能增强、中间件、日志装饰

### 10. 外观模式 (Facade)

- **文件**: `facade/main.go`
- **描述**: 为子系统中的一组接口提供一个一致的界面
- **应用场景**: API 网关、系统集成、复杂系统简化

### 11. 享元模式 (Flyweight)

- **文件**: `flyweight/main.go`
- **描述**: 运用共享技术有效地支持大量细粒度的对象
- **应用场景**: 对象池、缓存系统、游戏开发

### 12. 代理模式 (Proxy)

- **文件**: `proxy/main.go`
- **描述**: 为其他对象提供一种代理以控制对这个对象的访问
- **应用场景**: 远程代理、虚拟代理、保护代理、缓存代理

## 行为型模式 (Behavioral Patterns)

### 13. 责任链模式 (Chain of Responsibility)

- **文件**: `chain/main.go`
- **描述**: 将请求沿着处理者链传递，直到某个处理者处理它为止
- **应用场景**: 日志级别处理、审批流程、异常处理

### 14. 命令模式 (Command)

- **文件**: `command/main.go`
- **描述**: 将请求封装成对象，从而可以用不同的请求对客户进行参数化
- **应用场景**: 撤销/重做功能、宏命令、队列请求

### 15. 迭代器模式 (Iterator)

- **文件**: `iterator/main.go`
- **描述**: 提供一种方法顺序访问一个聚合对象中各个元素
- **应用场景**: 集合遍历、数据库结果集、文件读取

### 16. 中介者模式 (Mediator)

- **文件**: `mediator/main.go`
- **描述**: 定义一个中介对象来封装一系列对象之间的交互
- **应用场景**: 聊天室、智能家居控制、UI 组件通信

### 17. 备忘录模式 (Memento)

- **文件**: `memento/main.go`
- **描述**: 在不破坏封装性的前提下，捕获一个对象的内部状态
- **应用场景**: 撤销功能、游戏存档、事务回滚

### 18. 观察者模式 (Observer)

- **文件**: `observer/main.go`
- **描述**: 定义对象间的一种一对多的依赖关系
- **应用场景**: 事件系统、MVC 架构、发布订阅

### 19. 状态模式 (State)

- **文件**: `state/main.go`
- **描述**: 允许对象在内部状态改变时改变它的行为
- **应用场景**: 状态机、游戏 AI、工作流

### 20. 策略模式 (Strategy)

- **文件**: `strategy/main.go`
- **描述**: 定义一系列算法，把它们一个个封装起来，并且使它们可相互替换
- **应用场景**: 支付方式、排序算法、压缩算法

### 21. 模板方法模式 (Template Method)

- **文件**: `template/main.go`
- **描述**: 在一个方法中定义一个算法的骨架，而将一些步骤延迟到子类中
- **应用场景**: 数据处理流程、测试框架、代码生成

### 22. 访问者模式 (Visitor)

- **文件**: `visitor/main.go`
- **描述**: 表示一个作用于某对象结构中的各元素的操作
- **应用场景**: 编译器设计、文档处理、数据分析

## 使用说明

每个设计模式都包含：

- 完整的 Go 语言实现
- 详细的中文注释
- 实际应用场景示例
- 可直接运行的 main 函数

运行示例：

```bash
# 运行单例模式示例
go run designpattern/singleton/main.go

# 运行观察者模式示例
go run designpattern/observer/main.go

# 运行策略模式示例
go run designpattern/strategy/main.go
```

## 设计原则

这些实现遵循以下设计原则：

- **单一职责原则**: 每个类只有一个引起变化的原因
- **开闭原则**: 对扩展开放，对修改关闭
- **里氏替换原则**: 子类必须能够替换其基类
- **接口隔离原则**: 客户端不应该依赖它不需要的接口
- **依赖倒置原则**: 高层模块不应该依赖低层模块，都应该依赖抽象

## Go 语言特色

在 Go 语言中实现设计模式时，充分利用了 Go 的特性：

- **接口**: Go 的隐式接口实现使得代码更加灵活
- **组合**: 通过结构体嵌入实现继承的效果
- **并发**: 利用 goroutine 和 channel 实现并发模式
- **函数式**: 函数作为一等公民，支持函数式编程风格
